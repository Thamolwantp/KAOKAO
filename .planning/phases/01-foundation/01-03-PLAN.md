    - On unexpected error: log the error and exit(1)

    Install tsx as dev dependency if not already present: npm install --save-dev tsx.

    Create scripts/enable-rls.sql:
    Write one ALTER TABLE statement per table using the Prisma @@map() names (the actual PostgreSQL table names):
    ALTER TABLE users ENABLE ROW LEVEL SECURITY;
    ALTER TABLE pets ENABLE ROW LEVEL SECURITY;
    ALTER TABLE pet_photos ENABLE ROW LEVEL SECURITY;
    ALTER TABLE weight_logs ENABLE ROW LEVEL SECURITY;
    ALTER TABLE vaccinations ENABLE ROW LEVEL SECURITY;
    ALTER TABLE vet_visits ENABLE ROW LEVEL SECURITY;
    ALTER TABLE lost_cases ENABLE ROW LEVEL SECURITY;
    ALTER TABLE sightings ENABLE ROW LEVEL SECURITY;
    ALTER TABLE ai_scans ENABLE ROW LEVEL SECURITY;
    ALTER TABLE ai_matches ENABLE ROW LEVEL SECURITY;
    ALTER TABLE pet_embeddings ENABLE ROW LEVEL SECURITY;
    ALTER TABLE notifications ENABLE ROW LEVEL SECURITY;
    ALTER TABLE push_subscriptions ENABLE ROW LEVEL SECURITY;

    Each ALTER TABLE … ENABLE ROW LEVEL SECURITY is idempotent in PostgreSQL (safe to re-run).

    Execute the SQL via Prisma: npx prisma db execute --file scripts/enable-rls.sql --schema schema.prisma

    Alternatively, paste the SQL into Supabase Dashboard → SQL Editor and run it there if the CLI command fails.

    Verify RLS is enabled using a node script (prisma.$queryRaw returns actual rows, unlike db execute):
    node -e "
    const {PrismaClient}=require('@prisma/client');
    const p=new PrismaClient();
    p.\$queryRaw\`SELECT relname, relrowsecurity FROM pg_class WHERE relname IN ('users','pets','pet_photos','weight_logs','vaccinations','vet_visits','lost_cases','sightings','ai_scans','ai_matches','pet_embeddings','notifications','push_subscriptions') ORDER BY relname\`
    .then(rows=>{
      const allEnabled=rows.every(r=>r.relrowsecurity===true);
      console.log('RLS enabled on all 13 tables:', allEnabled);
      if(!allEnabled) rows.filter(r=>!r.relrowsecurity).forEach(r=>console.error('MISSING RLS:',r.relname));
      p.\$disconnect();
      if(!allEnabled) process.exit(1);
    }).catch(e=>{console.error(e);p.\$disconnect();process.exit(1);})
    "
