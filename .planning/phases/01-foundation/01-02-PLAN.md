## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Server components → Supabase API | Authenticated via anon key (RLS enforced) or service role key (bypasses RLS — server-only) |
| Client components → Supabase API | Authenticated via anon key only; service role key must never reach client |
| .env.local → git repository | .env.local must be git-ignored; only .env.example is committed |
| DATABASE_URL → Supabase PostgreSQL | Direct DB access via Prisma — never expose to client; server-side only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | SUPABASE_SERVICE_ROLE_KEY | mitigate | Import "server-only" at top of lib/supabase/service.ts — build error if imported in client bundle; key stored only in .env.local (git-ignored) |
| T-02-02 | Information Disclosure | DATABASE_URL in .env.local | mitigate | .gitignore covers .env.local; .env.example has placeholder values only; verify with `git ls-files .env.local` returns empty |
| T-02-03 | Elevation of Privilege | Prisma client used in client components | mitigate | lib/prisma.ts must only be imported in Server Components, Route Handlers, or server actions — no "use client" files may import it; enforced by TypeScript module boundary checks in Phase 2 |
| T-02-04 | Spoofing | DATABASE_URL with wrong credentials | accept | Low risk at foundation stage — only developer can set .env.local; migration failure is immediately visible |
| T-02-SC | Tampering | npm installs (@supabase/supabase-js, @supabase/ssr, @prisma/client, server-only) | mitigate | Verify packages on npmjs.com before install: @supabase/supabase-js (official Supabase package), @supabase/ssr (official Supabase SSR package), @prisma/client (official Prisma package), server-only (Next.js official utility); all have millions of weekly downloads |
